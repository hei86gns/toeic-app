# yuriaToeic_v1.html 修正項目リスト

見つかった不具合・改善点をここに書き溜める。まとめて対応するタイミングで着手する。

## 未対応

- [ ] **全パートが同程度の正答率だと、週次メニューの同じモジュールが連日並んでしまう**
  優先度が僅差の場合、モジュール選定ロジック（`pickModuleQueue`/`buildDayBlocks`）が同じモジュール（例：速度・実戦）を繰り返し選びやすい。特に自己診断（未経験者向け）で全パート同一の正答率を設定した場合に発生しやすい。今回の5件とは別件のため、今回は対応せず記録のみ。

## 対応済み

- [x] **週次メニュー画面に「ホーム／診断結果に戻る」導線がない**
  週次メニュー画面（`renderMenu`）に「診断結果に戻る」（`#/diagnosis`）リンクを追加。同じ根本原因（戻る導線の欠如）が診断結果画面（`renderDiagnosis`）にもあったため、そちらにも「最初から入力し直す」（`#/input`）リンクを追加。ブラウザ操作で両画面のリンクが正しく機能することを確認済み。

- [x] **単語クイズ・文法クイズができなくなった**
  実際にはクラッシュではなく、新しいホーム画面（曜日別表示）でその日のローテーションに語彙・文法が含まれないと入口が無くなっていたのが原因。ホーム画面に「語彙をやる」「文法をやる」の常設ボタンを追加し、その日の内容に関わらずいつでも語彙・文法に入れるようにした。

- [x] **最初の画面：TOEICを受けたことがない人向けの分岐**
  入力画面の前に「TOEICを受けたことがありますか？」の質問を追加。「はい」は従来通りパート別正答率の入力へ、「いいえ」は自己診断（ネイティブ/ビジネス/日常会話/初歩的/ほとんど話せないの5段階＋英検級・学習経験は参考情報として収集）に進み、選んだレベルに対応する目安スコア（950/730/550/450/300点）を精度換算した数値を全パートに均等に設定して診断する。

- [x] **文法説明のレイアウトを見やすく**
  解説文を句点で分割し、箇条書き（`.explanation-list`）で表示するように変更。

- [x] **文字サイズを変更できる設定メニュー**
  ホーム画面に⚙️アイコンを追加し`#/settings`画面を新設。「小・標準・大・特大」からフォントサイズを選べ、選択内容はlocalStorageに保存され次回起動時にも反映される（CSS全体をpx指定からrem指定に変更し、ルートのフォントサイズ変更で全体が連動して拡大縮小するように調整）。

- [x] **クイズの途中やめボタン**
  語彙クイズ・文法クイズの両方に「✕ 途中でやめる」ボタンを追加。押すと進行中のセッションを破棄し、語彙メニュー／該当の文法ユニット画面に戻る。
