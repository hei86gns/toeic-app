## この文書の前提・方法論

- **公式問題集は分析していません。** ETSの公式問題集は著作権保護された市販コンテンツであり、手元にもなく、Web上で探すこともできません（本アプリの設計仕様書§1が明記する「TOEIC公式問題は載せない」という方針とも整合）。
- 代わりに、**標準的な英文法の体系知識**＋TOEIC対策サイトの一般的な傾向情報を組み合わせて、各項目をサブカテゴリまで分解しました。
- 他のAIから共有いただいた12項目の表とも突き合わせ、**「主語と動詞の一致」が丸ごと抜けていた**こと、**「前置詞」が独立ジャンルとして弱かった**ことが分かったため、今回追加・強化しました（下記「他AI表との照合」参照）。

- 各項目に **難易度**（英文法としての複雑さ）と **TOEICでの位置づけ**（一般的なTOEIC対策書での扱われ方）を併記しますが、公式統計ではなく目安です。

## 1. 品詞の識別（実装済み：g_pos）

- 名詞・動詞・形容詞・副詞の識別（語尾のパターン：-tion/-ment/-ity/-ance/-ence=名詞、-ive/-able/-al/-ous/-ful=形容詞、-lyは副詞）
- **位置で見抜くヒント**：冠詞（a/an/the）や所有格（my/our/their）、前置詞の直後→名詞／名詞を修飾する位置→形容詞（例：an efficient process）／形容詞・動詞・文全体を修飾する位置→副詞（例：currently available、will soon finish）
- **派生語系列の識別**（例：success（名詞）/ successful（形容詞）/ successfully（副詞）/ succeed（動詞）のように、1つの語根から複数の品詞が派生する語群を見分ける）※他AI表の例をそのまま採用
- 名詞 vs 動名詞 vs 現在分詞の紛らわしさ（-ing形が3通りの働きを持つ）
- 紛らわしい語形（economic/economical、respectful/respective等）
- 複合名詞（名詞＋名詞の連続。例：employment policy）
- 難易度：基礎／TOEICでの位置づけ：**最頻出**。他AIの情報では「Part5出題数の約3割を占める最重要項目」との指摘あり（公式統計ではなくTOEIC対策界隈での目安）

## 2. 時制（実装済み：g_tense、要拡張）

- 基本時制：現在形・過去形・未来形（will / be going to）
- 進行形：現在進行形・過去進行形・未来進行形
- **完了形（サブカテゴリ）**：
  - 現在完了−継続（have + pp + since/for〜）
  - 現在完了−経験（have + pp + ever/never/before）
  - 現在完了−完了・結果（have + pp + already/yet/just）
  - 過去完了（had + pp、"by the time"等と結びつく）
  - 未来完了（will have + pp、"by + 未来の時"と結びつく）
  - 完了進行形（have been -ing / had been -ing）
- 時を示す語句との呼応（yesterday→過去、since 2015→現在完了、by next month→未来）
- **時制キーワード早見表**：現在進行・未来→currently, soon, next week／過去→yesterday, recently（過去形または現在完了形と結びつく点に注意）／現在完了→since〜, for the past 3 years
- 難易度：基礎〜中級（完了形の使い分けは中級）／TOEICでの位置づけ：最頻出

## 3. 態（✅実装済み：g_voice）

- 能動態 vs 受動態の基本
- 受動態＋by句の省略
- 群動詞の受動態（be looked at by〜、be taken care of〜）
- 使役・知覚動詞が受動態になると原形不定詞がto不定詞に変わる点（§9参照）
- 進行形受動態・完了形受動態（is being done、has been done）
- **見分けの実践ヒント**：空所の直後に目的語（名詞）が続いていれば能動態、前置詞句や何も続かなければ受動態（be+過去分詞）になりやすい
- 難易度：基礎〜中級／TOEICでの位置づけ：最頻出

## 4. 主語と動詞の一致（✅実装済み：g_agreement）※今回新規追加・重要な抜けだった項目

- 基本：単数主語→単数動詞、複数主語→複数動詞
- **集合名詞**（group / team / family等）：「まとまり」として見るか「個々のメンバー」として見るかで単数・複数が変わる
- **見かけ複数の学問名**（economics / politics / physics等）：形は複数だが単数扱い
- **each / every**：意味的には複数を連想させるが単数扱い（例：Each of the members is 〜／Each employee has 〜）
- **both A and B**：常に複数扱い
- **either A or B / neither A nor B / not only A but also B**：動詞はBの方（近い方の名詞）に一致させる
- **数量形容詞と名詞の数の一致**（a number of + 複数名詞+複数動詞 vs the number of + 複数名詞+単数動詞、等）
- **主語と動詞の間に修飾語句・挿入句が入るケース**（前置詞句や関係詞節が間に挟まり、直前の名詞につられて動詞の数を間違えやすい。文の核となる主語を見抜く必要がある）
- **動名詞・to不定詞・名詞節が主語の場合は単数扱い**（§8・§13との関連）
- 難易度：基礎〜中級（ルール自体は単純だが、修飾語句に惑わされるひっかけが多い）／TOEICでの位置づけ：最頻出とされる項目の一つ

## 5. 前置詞（✅実装済み：g_prepositions）※今回新規独立項目として強化

- **時間の前置詞**：at（時刻）／on（曜日・日付）／in（月・年・季節等の広い期間）
- **期限 vs 継続**：by（〜までに、期限）vs until（〜まで、継続）
- **期間の表し方**：for + 数値期間（for three years）vs during + 特定の期間を表す名詞（during the meeting）vs since + 起点（現在完了と結びつく）
- **between vs among**：between=2者間・特定数、among=不特定多数の間
- **despite / in spite of vs although**（前置詞 vs 接続詞、§6・旧g_prep_conjと関連）
- **場所の前置詞**：at（地点）／on（面・線上）／in（範囲・内部）
- **動詞・形容詞・名詞＋前置詞の決まったコロケーション**（depend on, consist of, be responsible for, in charge of等。§16語法・コロケーションとも重複するが、前置詞学習の核となる部分）
- 難易度：基礎〜中級（意味の使い分けは体系的だが、コロケーションの丸暗記が必要な部分は中級〜）／TOEICでの位置づけ：最頻出。Part5では前置詞単体の穴埋めも定番パターン

## 6. 接続詞（✅実装済み：g_prep_conj［前置詞vs接続詞］＋g_conjunctions［従属・相関］）

- 等位接続詞（and / but / or / so / for / yet）
- 従属接続詞：時（when / while / before / after / until）、条件（if / unless / **provided that** / **as long as** / **in case** / once）、理由（because / since / as）、譲歩（although / though / even though）
- **前置詞 vs 接続詞の対応表**：譲歩＝despite/in spite of（前置詞）vs although/even though（接続詞）／期間＝during（前置詞）vs while（接続詞）／理由＝because of・**due to**（前置詞）vs because・since（接続詞）
- 相関接続詞（both A and B / either A or B / neither A nor B / not only A but also B）※§4主語動詞の一致とも関連
- 前置詞 vs 接続詞 vs 副詞の三すくみ（because of vs because vs therefore等）
- 難易度：基礎〜中級／TOEICでの位置づけ：最頻出

## 7. 関係詞（✅実装済み：g_relative）

- 関係代名詞：主格（who/which/that）、目的格（whom/which/that、しばしば省略）、所有格（whose）、what（先行詞を含む）、複合関係代名詞（whoever/whatever/whichever）
- 関係副詞：where/when/why/how（前置詞＋whichへの書き換え可）
- 制限用法 vs 非制限用法（コンマの有無）
- 関係詞の省略（目的格関係代名詞、〈主格関係代名詞＋be動詞〉）
- **選び方の構造的な判別法**：①先行詞が人かモノ・事かで who系／which・that系を選ぶ、②後続の節が主語・目的語・所有格のいずれかが欠けた「不完全な文」なら関係代名詞、③後続が欠けのない「完全な文（S+V+O）」なら関係副詞（where/when等）
- 難易度：中級／TOEICでの位置づけ：頻出

## 8. 準動詞（✅実装済み：g_verbals［to不定詞・動名詞］＋g_participles［分詞・分詞構文］）

- to不定詞：名詞的用法・形容詞的用法・副詞的用法
- 動名詞：主語・目的語・前置詞の目的語になる（§4：動名詞主語は単数扱い）
- 動名詞 vs to不定詞を目的語に取る動詞の区別（remember to do vs remember doing、stop to do vs stop doing）
- **分詞形容詞（現在分詞=能動 vs 過去分詞=受動）**：leading companies（業界をリードしている＝能動）vs attached documents（添付された＝受動）のように、修飾する名詞が「する側」か「される側」かで使い分ける
- **感情動詞の分詞形容詞**：interested vs interesting、excited vs exciting、confused vs confusingのように、人が主語（感情を「受ける」）なら-ed、モノ・事が主語（感情を「引き起こす」）なら-ingになる典型パターン
- 分詞構文（付帯状況・理由・時など）、独立分詞構文
- 難易度：中級〜上級（分詞構文は難所）／TOEICでの位置づけ：頻出（不定詞・動名詞の基本、感情動詞の分詞形容詞）〜低頻度（分詞構文）

## 9. 使役動詞・知覚動詞（✅実装済み：g_causative）

- 使役動詞：make/have/let + O + 原形不定詞
- get + O + to不定詞（例外的にto不定詞）
- help + O + (to) do
- 知覚動詞：see/hear/feel/watch + O + 原形/現在分詞
- 受動態になると原形不定詞がto不定詞に変わる（He was made to work.）
- 難易度：中級／TOEICでの位置づけ：中頻度だが典型パターン

## 10. 比較（✅実装済み：g_comparison）

- 原級（as ~ as）、比較級（-er/more~than）、最上級（the -est/the most~）
- 比較級の強調（much/far/even/a lot + 比較級）
- the + 比較級, the + 比較級（〜すればするほど…）
- クジラ構文（no more ~ than…、低頻度）
- 難易度：基礎〜中級／TOEICでの位置づけ：頻出

## 11. 代名詞（✅実装済み：g_pronouns）

- 人称代名詞の格（主格・目的格・所有格・所有代名詞）。特に**「所有格＋名詞」の組み合わせ**（their report等）が頻出パターン
- 再帰代名詞（oneself）：主語と目的語が同じ場合（He introduced himself）、「〜自身で」の強調（by himself = on his own）
- 指示代名詞（this/that/these/those）
- 不定代名詞：one/another/other/others/the other(s)、each/every、all/most/some/any/none
- 代名詞と先行詞の数・性の一致
- 難易度：基礎〜中級／TOEICでの位置づけ：頻出、格変化の基礎問題として定番

## 12. 助動詞（✅実装済み：g_modals）

- 基本助動詞：can/could/may/might/must/should/will/would
- 助動詞相当表現：have to/be able to/be going to/ought to
- 〈助動詞＋have＋過去分詞〉（should have done/must have done/may have done）
- 提案・要求のshould（§14仮定法現在と関連）
- 難易度：基礎〜中級／TOEICでの位置づけ：頻出

## 13. 名詞節（✅実装済み：g_noun_clauses）

- that節（主語・目的語・補語。しばしば省略される）
- wh節（what/who/where/when/why/how + S+V）
- whether/if節（「〜かどうか」）
- 同格のthat節（the fact that〜等）
- 難易度：中級／TOEICでの位置づけ：中頻度

## 14. 仮定法・条件文（全面的に細分化・前回修正済み）

### 14-1. 直説法の条件文（基礎・最頻出）
If + 現在形, will/can/should + 動詞原形 ／ 難易度：基礎／頻度：最頻出

### 14-2. 仮定法過去（現在の事実に反する仮定）✅実装済み：g_conditionals_advanced
If + 過去形, would/could/might + 動詞原形（be動詞はwereが伝統的）／ 難易度：中級／頻度：中

### 14-3. 仮定法過去完了（過去の事実に反する仮定）✅実装済み：g_conditionals_advanced
If + had + pp, would/could/might + have + pp ／ 難易度：中級／頻度：中

### 14-4. 混合型仮定法（Mixed Conditionals）✅実装済み：g_conditionals_advanced
過去の条件→現在の結果（If + had + pp, would + 動詞原形）／ 難易度：上級／頻度：低

### 14-5. 仮定法現在（提案・要求・必要性のthat節）✅実装済み：g_mandative ※重要度が高い
suggest/recommend/propose/require/insist/demand + that + S + 動詞原形、It is essential/necessary + that + S + 動詞原形。ビジネス英語の定型表現として頻出、「900点専用」ではない ／ 難易度：中級／頻度：中〜高

### 14-6. If節の省略・倒置（フォーマル）✅実装済み：g_subjunctive_extra
Were I ~, / Had I known ~, / Should you need ~, ／ 難易度：上級／頻度：低

### 14-7. wish / as if・as though + 仮定法 ✅実装済み：g_subjunctive_extra
I wish + 仮定法過去/過去完了、as if/as though + 仮定法 ／ 難易度：中級〜上級／頻度：低

### 14-8. その他の仮定法的表現 ✅実装済み：g_subjunctive_extra
without/but for/otherwise + 仮定法、if only ／ 難易度：上級／頻度：低

## 15. 数量表現（✅実装済み：g_quantifiers）

- 可算名詞・不可算名詞の区別
- much/many、few/little、a few/a little（肯定・否定のニュアンス差）
- each/every（単数扱い、§4と関連）、all/most/some/any/none
- a number of（複数扱い）vs the number of（単数扱い）※§4と関連
- 難易度：基礎〜中級／TOEICでの位置づけ：中頻度

## 16. 倒置・強調構文（✅実装済み：g_inversion）

- 否定の副詞（Never, Rarely, Not only等）が文頭に来た場合の倒置
- Only + 副詞句が文頭に来た場合の倒置
- 強調構文（It is ~ that ...）
- 難易度：上級／TOEICでの位置づけ：低頻度

## 17. 語法・コロケーション（✅実装済み：g_collocations）

- 動詞＋前置詞の決まった組み合わせ（depend on, consist of, comply with等。§5前置詞と重複）
- 紛らわしい類義語の使い分け（during vs while, despite vs although, raise vs rise, affect vs effect等）
- 難易度：中級／TOEICでの位置づけ：頻出（語彙寄りだが文法問題としても出る）

## 出題形式の補足：「動詞の形の識別」という複合パターン

他AI表の「2. 動詞（原形・ing・過去分詞）」は、実は単一の文法概念ではなく、**時制（§2）・態（§3）・準動詞（§8）の知識を組み合わせて答える出題形式**です（例：4択が increase/increasing/increased/to increaseのように同じ語の変化形で並ぶパターン）。独立した文法カテゴリとしてではなく、複数分野を横断した「出題パターン」として文法クイズの応用問題に位置づけるのが妥当です。
