# TOEIC対策アプリ v1 設計仕様書

**目的**：ユーザーのパート別正答率から弱点を診断し、研究に基づく週次学習メニューを自動生成する。
**v1**：ゆりあ専用・単一HTMLファイル＋localStorage。
**将来**：configとコンテンツを分離し、Supabaseへ載せ替えて汎用マルチユーザー化。
作成日：2026年7月

---

## 1. 基本方針

- 診断ロジックはルールベース（外部AI不要）。心臓部は「優先度＝問題数 ×（目標−現在）」。
- **設定・コンテンツ・ロジックを分離**する。閾値・目標・モジュール内容はすべて `config` に置き、汎用化時にコードを触らず調整できるようにする。
- 著作権：**TOEIC公式問題は載せない**。本アプリが提供するのは「診断・学習メニュー・方法・進捗管理」まで。語彙はCC BY-SA 4.0の公開リスト（NGSL・TSL）由来の自作データのみ。

---

## 2. データモデル

localStorage に1ユーザー分を保存（v1）。将来はこの構造を配列化してマルチユーザーへ拡張。

```json
{
  "user": {
    "name": "ゆりあ",
    "parts": {
      "P1": 1.00, "P2": 0.72, "P3": 0.44, "P4": 0.67,
      "P5": 0.47, "P6": 0.44, "P7A": 0.66, "P7B": 0.24
    },
    "goalMode": "fixed",        // "fixed" | "score"
    "goalValue": 0.85,          // fixedなら目標正答率、scoreなら目標スコア(例600)
    "timeBudgetMin": 300,       // 週あたり学習時間(分)
    "flags": { "p7bTimeSuspect": true, "foundationGate": true }
  },
  "history": [
    { "date": "2026-07-20", "parts": { }, "note": "" }
  ]
}
```

- `parts` は 0〜1 の正答率。入力UIでは％で受け、内部で小数に変換。
- `config`（問題数・閾値・変換表・モジュール・メニュールール）は**コードと別オブジェクト**に置く（§9）。

---

## 3. 診断エンジン（優先度式）

**心臓部：**

```
優先度 = 問題数 × (目標正答率 − 現在正答率)
```

負になる（＝すでに目標超え）パートは優先度0扱い。

### 3-4. 検算（ゆりあさんの実データ／目標85%固定の場合）

| パート | 問題数 | 正答率 | 優先度 |
|---|---|---|---|
| Part 3 | 39 | 44% | **16.0** ① |
| Part 7B | 25 | 24% | **15.3** ②※ |
| Part 5 | 30 | 47% | **11.4** ③ |
| Part 6 | 16 | 44% | 6.6 |
| Part 7A | 29 | 66% | 5.5 |
| Part 4 | 30 | 67% | 5.4 |
| Part 2 | 25 | 72% | 3.3 |
| Part 1 | 6 | 100% | 0 |

②の※はフラグ（下記）による分岐対象を示す。

---

## 4. 特別ルール（フラグ）

- **7B時間切れ疑いフラグ**：`P7A − P7B ≥ 25%` で点灯。ゆりあさんは42%で**発火**。
- **土台ゲート**：`P5 < 閾値` または `P6 < 閾値`（例：閾値55%）で発火。ゆりあさんはP5 47%／P6 44%で**発火**。

閾値はすべて `config` 値。

## 5. モジュール表（パート → 処方）

| モジュール | 対象パート | 処方 | コンテンツの持ち方 | 主なKPI |
|---|---|---|---|---|
| 音声知覚 | P3, P4 | シャドーイング（精聴→スクリプト付→無）＋設問先読み | 方法提示のみ | シャドーイング素材数／週 |
| 定型応答 | P1, P2 | 頻出パターン反復（維持中心） | 方法提示のみ | 正答率の維持 |
| 文法 | P5, P6 | 文法ユニット解説＋4択クイズ | **アプリ内実装**（§11-2） | ユニット完了数、クイズ正答率 |
| 語彙 | P5〜P7 | レベル別単語をフラッシュカード／クイズで練習 | **アプリ内BUILTIN**（§11-1） | クイズ実施回数 |
| 精読 | P7A, P7B | 精読＋情報照合の練習 | 方法提示のみ | 精読文書数／週 |
| 速度・実戦 | P5→P7 | 多読＋時間を計った通し演習 | 方法提示のみ | 通し演習回数 |

## 6. 週次メニュー生成手順

1. **フェーズ判定**：土台ゲート発火 → 「土台フェーズ」。それ以外は優先度上位で「処理力／速度フェーズ」。
2. **モジュール選定**：優先度ランキング上位＋発火フラグに対応するモジュールを抽出。
3. **曜日割り**：`timeBudgetMin` を曜日に配分。各ブロックにモジュール・方法・KPIを割り当て。
4. **出力**：曜日ごとの学習ブロック（内容＋所要目安＋KPI）を提示。

### 6-1. 生成例（ゆりあさん／土台フェーズ発火時・1週間）

| 曜日 | 内容 | ねらい |
|---|---|---|
| 月 | 文法ユニット（品詞の識別）解説＋クイズ ＋ 語彙10問 | 土台（文法・語彙） |
| 火 | Part3 シャドーイング（精聴→スクリプト付） | 音声知覚 |
| 水 | 語彙10問（復習）＋ 文法ユニット（時制）クイズ | 土台 |
| 木 | Part3 シャドーイング（スクリプト無まで） | 音声知覚の自動化 |
| 金 | 文法ユニット（前置詞vs接続詞）＋ 語彙10問 | 文法＋語彙 |
| 土 | Part7A 精読 ＋ 7B切り分け診断（無制限リテスト） | 読解＋時間問題の切り分け |
| 日 | 予備／通し演習（時間配分の練習）または休養 | 実戦感覚 |

各ブロックは5〜15分で完結する通勤スマホ向け粒度。

## 7. 目標モード（fixed / score）と 7B分岐

### 7-1. 目標モード切替
- `"fixed"`：全パート共通の目標正答率（既定85%）。
- `"score"`：目標スコア（例600）→ 必要正答率へ変換。変換表は `config` 上の調整可能な目安値。

### 7-2. 7B分岐
- **既定：自己申告**（「Part7Bを解き切れた？」を軽く尋ねる）。
- **精密診断：無制限リテスト**（「精密診断」ボタン）。P7Bのみ時間無制限で解き直し、正答率が上がれば「時間切れが原因」、変わらなければ「情報照合が苦手」と判定。

## 8. 画面構成（v1）

1. **入力**：8パート正答率 ＋ 目標モード切替 ＋ 目標値 ＋ 時間予算。
2. **診断結果**：優先度ランキング表＋フラグ表示＋7B分岐質問。
3. **週次メニュー**：曜日割り＋各ブロックの方法・KPI。文法→アプリ内ユニット、語彙→アプリ内単語画面への導線。
4. **文法コンテンツ**：ユニット一覧 → 解説 → 4択クイズ → 結果。
5. **語彙コンテンツ**：アプリ内フラッシュカード／10問クイズ（BUILTINデータから出題）。

通勤スマホ利用が主用途のため、全画面をモバイル幅（縦持ち・片手操作）前提でレイアウトする。

v2追加：メニューチェック／履歴グラフ／Supabase同期。
v3候補：スコア写真をVision APIで自動入力。

## 9. 汎用化のための設計上の約束

- `config` をコードから完全分離。
- 永続化層を `save()/load()` で抽象化し、localStorage↔Supabaseを差し替え可能に。
- ユーザーを配列で持てる構造にしておく（v1は1件のみ）。
- モジュールの `method` 文言はデータ側に置き、内容改訂をコード変更なしで行えるように。

## 11. コンテンツ詳細

### 11-1. 語彙モジュール（アプリ内BUILTIN）

**方針（確定）**：tangochoとは連携しない。TSL 1,258語の日本語訳データをアプリのBUILTINとして内蔵し、アプリ内でフラッシュカードと10問クイズを提供する。

**経緯**：
- tangochoの2,273語を分析したところ、TSL（TOEIC特化1,258語）との重複がわずか12語（0.5%）であり、tangochoは遺伝・医学・経済などの上級・専門語が中心でTOEIC学習に不適と判明。
- tangochoの「マイ単語を読み込む」機能はオーナーのアカウントに紐づくため、ゆりあ専用アプリのコンテンツ配布手段として不適。
- 以上から、TSLデータをアプリBUILTINとして持つ方針に確定。

**語彙データ**：
- TSL 1.1（Browne & Culligan、CC BY-SA 4.0）1,258語に独自日本語訳を付与。
- 形式：`[{"w": "word", "m": "日本語訳"}, ...]`
- ファイル：`toeic_tsl_full_tangocho.json`（Claude Codeへ引き継ぐ）

**将来**（v2以降）：
- NGSL T1（最頻1,000語）の日本語訳を追加し、土台語彙を補強する。
- tangochoとのディープリンク連携は、アプリ内語彙とは別に「発展語彙（上級）」として検討余地あり。

**UIに必要な機能**：
- フラッシュカードモード（英→日、タップでめくる）
- 10問クイズモード（4択）
- 正誤履歴を保存し、苦手語を優先出題

### 11-2. 文法モジュール（アプリ内新規）

**方針**：TOEICアプリの独自価値としてアプリ内に実装。

- **データ構造**：文法ユニットの配列。各ユニットに `title` / `explanation` / `quiz`（4択×複数問）。
- **クイズ形式**：短文4択穴埋め（Part5型）。例文・解説はすべてオリジナル。
- **初期スコープ**：品詞の識別、時制、態、前置詞vs接続詞、関係詞、準動詞。
- **診断連動**：P5/P6が弱い場合、該当ユニットを週次メニュー・ユニット一覧の上位に出す。
- **粒度**：1ユニット＝解説+クイズで5〜10分。

```json
{
  "grammarUnits": [
    {
      "id": "g_pos",
      "title": "品詞の識別",
      "explanation": "（オリジナル解説）...",
      "targetParts": ["P5", "P6"],
      "quiz": [
        {
          "q": "The report was ___ reviewed.",
          "choices": ["care", "careful", "carefully", "caring"],
          "answer": 2,
          "note": "動詞を修飾＝副詞"
        }
      ]
    }
  ]
}
```
